# Entrepreneur Review 创业复盘

我认为，创业是一种资源极其有限的情况下，将知识快速变现的游戏。

## 我的角色

我在团队中的角色是前端工程师、前端组长、MCP 服务维护者。

Vue、React、Uniapp 我都干过了。常见的大厂组件库 Tdesign、Ant design、ChatUI 我都使用过。甚至 Next.js 这种全栈框架也用过。我还用 Shadcn/UI 结合 AI 写了小项目。现在我维护着许多团队的前端项目，都属于 Web 项目。

我最多曾经带着 3 名前端实习生，用 uniapp 开发小程序。中途我把 Uniapp 小程序项目移交出去，进度由项目经理把握。后来一个到期离职，另一个因为家庭原因暂时退出，还有一个转全栈了。

最近还多了一个记忆引擎的 MCP 服务维护者身份，负责把写好的记忆服务包裹成 LLM 可以直接调用的服务，这是一种新的产品形态，直接面向开发者。可以发布到各大开发平台的 MCP 市场，用户通过订阅获得密钥，然后开发者就可以持续收费，躺着挣钱。
## 我得到的教训

### 研发

谨慎使用 Uniapp 开发。

尽管它声称通过编译可以实现“一次开发，到处运行”，但实际开发过程中发现，用它写的小程序实在是难以维护。

这主要是因为 H5、微信开发者工具、各个手机客户端都有不同的渲染逻辑，注意，安卓、iOS、鸿蒙也有不同的渲染逻辑！

这意味着任何一个交互程度略复杂的组件，都要花大量时间精力来打磨，来适配不同终端。而当组件出现意料之外的行为，也很难判断和底层渲染逻辑有什么关系，只能靠 AI 一遍一遍抽卡修复。

因此我明白为什么 Uniapp 的组件生态很少，而且都很简单。怪不得以前听说 Uniapp 只能用来做玩具。

### 管理

进度管理永远是一件难事。

假如你是给工作项/需求排期的人，那么你不但要知道谁（研发）来实现需求，而且还要了解是什么水平的研发，实现什么样的需求。这样才能给合适的人安排合适的需求，保证迭代/项目按时交付。

所以你一方面要关注研发的能力，另一方面要关注需求的复杂度。

你要知道，研发的基础能力包括对需求的理解能力，还有及时对齐理解的能力。也就是说，拿到一个需求描述之后，不论是口头的还是书面的，研发是否能迅速拆解成清晰的步骤，是否能对模糊的部分提出疑问，并且主动去找人对接。对技术的掌握倒是其次，因为 AI 的能力越来越强。

麻烦就麻烦在，基础能力是变化的。研发可以从过往实现中学习成长，从而越来越熟练，效率越来越高；但也可能因为没有及时积淀，导致能力滑坡。你要及时跟进研发的能力状态。

进一步，更高的要求是，用优雅的方法而非复杂的方法实现需求。

让我们来看需求吧。需求的复杂度倒是不会因为熟练程度而变化，但需求有合理与否的区别。这是因为资源并非无限，你需要回答如何在时间有限、研发力量有限的情况下，到底哪些需求值得当下去做。

这就是一个决策问题，你会使用何种方式来判断，来排序？这就是你要精进的技能点。

很遗憾，我不认为现在团队里有人点出了进度管理的高级技能点。

---

职责太多不是好事。

进度管理通常是项目经理的职责，而项目经理是一种角色，当有专人做项目经理时，这就成了一种职位。

但初创公司的资源非常有限，大家都希望尽量用更少的人做更多的事，于是大家通常都是多面手，也就是一个人承担多个角色的职能。比如项目经理同时是研发人员，或者产品经理同时是市场营销，还有 CEO 兼任了财务。

问题在于，如果一个人身上集中了太多的职责，比如 CEO 兼任了所有的高管职位，那会带来灾难的。

拿我自己来说，首先是我自己的作息无法保证。我必须牺牲部分睡眠时间才能处理这么多事务，这会导致作息混乱，影响我的身心健康。比如心脏、肝脏受损，甚至是鼻炎。长期下去，我的活力会慢慢丧失，身体机能下降，内分泌失调。例如熬灯夜战时，皮质醇持续分泌，导致我误以为夜里效率高。

同时，我每天还要和各种职能的小伙伴对接，于是团队沟通就会变得非常低效。每天有十几个人给我发工作消息，而别人不知道我什么时候在线，如果我凌晨3点才睡，下午下午1点才醒，而其他人作息是正常的。那么我收不到上午的消息，只能下午回复消息——我们失去了半天的沟通窗口！

更麻烦的是，我也不能保证每天都是如此，因为有时候我要在白天见客户，下午补觉。而且我也不能要求别人的作息和我一样，这不现实。我只能在约定的会议时间内尽量上线，同时一有空就处理工作消息（异步）。

这时候我要怎么办呢？每天都把自己的时间窗口列出来，告诉团队小伙伴？问题是困意袭来的时候我根本控制不住自己的眼皮。还有一个办法就是放权，但是我放权之后只是减少了沟通次数，仍然需要和组长沟通呀！

还有一点，我不可能在所有领域都有很高的决策水平，尽管我是创始人，我也不可能面面俱到（否则就不需要团队了）。当我活力逐渐丧失，每天要和海量的人打交道，再加上和 AI 聊天久了，我会认为一两句话描述需求和下判断是 ok 的！

到最后，我难以做出任何长远决策，哪怕是下一周该干什么，都以“没有时间“来逃避。

## 我得到的经验

1. 主动出击，推销自己。恰恰是因为一个人做不成大生意，所以有无数的团队渴求人才。而每个人的简历上肯定都有一两个闪光点，遇到气味相投的团队时，把最好的一面拿出来，相信自己！
2. 识别他人的能力，把团队成员的能力看成一种资源。假如你已经有团队了，一定一定要把团队的能力转化为自己的能力。无论是写文档、写代码，还是沟通协作。
## 总结

我在技术方面已经走得够远了，是时候把技术变现了。

当我认为我不能从团队中学到更多时，我就可以抽身了。
